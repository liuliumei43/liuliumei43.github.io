实验报告：利用云平台搭建个人博客

山东大学 计算机科学与技术 学院

云计算技术 课程实验报告

学号：202202130290 姓名：叶鑫 班级：22数据
实验题目：利用云平台搭建个人博客
实验学时：2 实验日期：2025/3/12

实验目的：本实验旨在通过实践操作，掌握基于云计算环境的网站部署技能。

具体包括：构建环境并实现个人博客/网站的搭建，撰写实验报告，并将这份报告作为一篇博文发布在自己的博客 / 网站上。

硬件环境： 

联网的计算机一台

软件环境：

Windows or Linux

实验步骤与内容：

一． 准备工作

  从官网下载了node.js、git以及hexo，以及注册github账号。

 

 

 

 使用管理员打开powershell，通过npm install -g hexo-cli下载，并使用hexo -v查看是否是否安装成功。

二、 网站初始化：

 可以新建一个文件夹，里面存放网站的各种文件。可以通过cmd或者打开git bush使用hexo init进行网站的初始化。随后可以使用hexo g进行生成网站静态页面，hexo s可以在本地部署网站，http://localhost:4000/。

 

可以尝试hexo new “helloworld2”new一个新博客

期间可以会遇见报错，可以根据错误提示，使用 npm install –force’强制安装所需的依赖库，如果没权限安装，可以在管理员模式下的powershell进行安装。

三、 github ssh配置

  首先创建一个新仓库，仓库名：username.github.io

 

 如果先前就配置过git，可以使用git config –list查看用户名和邮箱地址是否与github对应。不是的话，使用git config –global user.name “yourname”和git config –global user.email “youremail”。

 

 然后生成ssh密钥，并指定保存地址。

 

 在github上，在settings中的SSH和GPG keys上添加一个新的ssh密钥。将刚刚保存在指定位置上的id_rsa.pub的密钥复制粘贴在GitHub上即可。

 

 

 

 然后修改文件夹下的config.yml，将deploy修改成如下，填写你网页仓库地址。

 

 

 

四、 测试：

 在git bush下使用npm install hexo-deployer-git –save下载推送工具，随后就可以使用hexo d将网站推送到github上的仓库。然后就可以在项目的设置里找到你的博客地址

 

 如果重新修改你的网站要再次部署可以输入hexo clean && hexo g && hexo d

 这是一个部署的标准的三连命令，用于清理、生成和部署 Hexo 博客

 

结论分析与体会：

通过本次实验，我成功利用云计算平台搭建了一个个人博客网站，并掌握了基于Hexo框架的网站部署流程。我学会了如何使用Hexo框架快速生成静态网站。Hexo的简洁性和高效性让我印象深刻。总的来说，本次实验不仅让我掌握了基于Hexo的网站部署技能，还让我对云计算技术有了更深的理解。
